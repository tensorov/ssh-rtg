# Security Policy

## Supported Versions

Only the latest release is currently supported. When v0.1.0 is published, the
latest tagged release will receive security updates. Older versions are not
supported and users are encouraged to upgrade.

## Reporting a Vulnerability

Please report security vulnerabilities to **security@tensorov.com**.

Include the following in your report:
- A description of the vulnerability and its impact
- Steps to reproduce (proof of concept is ideal)
- Affected versions and configurations
- Any suggested fix, if known

You will receive an acknowledgment within 48 hours. We aim to triage and
release a fix within 90 days of confirmation. If the vulnerability is
accepted, a security advisory will be published on GitHub. If declined, a
clear explanation will be provided.

## Disclosure Policy

We follow a coordinated disclosure process:
1. Reporter submits vulnerability via email
2. We confirm receipt and begin triage (within 48h)
3. A fix is developed and tested
4. The fix is released and a public advisory is published
5. Reporter is credited (if desired) in the advisory

Thank you for helping keep this project and its users safe.
